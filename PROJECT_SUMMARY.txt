================================================================================
                              DATABASE SCHEMA
================================================================================

TABLES:
  1. projects
     - id (UUID, Primary Key)
     - name (TEXT)
     - description (TEXT)
     - created_at (TIMESTAMPTZ)
     - updated_at (TIMESTAMPTZ)

  2. code_files
     - id (UUID, Primary Key)
     - project_id (UUID, Foreign Key → projects.id)
     - name (TEXT)
     - content (TEXT)
     - language (TEXT)
     - created_at (TIMESTAMPTZ)
     - updated_at (TIMESTAMPTZ)

SECURITY:
  - Row Level Security (RLS) enabled
  - Public access policies configured
  - Cascade delete on project removal

================================================================================
                              NOTES
================================================================================

DESIGN DECISIONS:
  - Monaco Editor chosen for professional code editing experience
  - Supabase for reliable persistent storage
  - eval() used for browser-based JavaScript execution
  - Console override for output capture
  - Debounced auto-save to reduce database writes
  - Resizable panels for customizable workspace

SECURITY CONSIDERATIONS:
  - Code execution sandboxed in browser
  - No server-side code execution
  - Public database access (no authentication required)
  - RLS policies ready for future authentication

PERFORMANCE OPTIMIZATIONS:
  - Debounced auto-save (1 second)
  - Lazy loading of Monaco Editor
  - Database indexes for faster queries
  - Efficient React rendering with proper hooks

FUTURE ENHANCEMENTS:
  - Multiple language support (Python, C++, etc.)
  - Code sharing with unique URLs
  - Version history
  - Real-time collaboration
  - Debugging tools
  - Code snippets library
